# How the BracketIQ Slate Model Generates Its Output

This document describes how the daily slate JSON is produced: what inputs are used, how spread/moneyline/over-under edges are computed, and what each field in the output means.

---

## Overview

The slate is generated by **`scripts/slate_today.py`** (in this repo). It runs as part of the **BracketIQ Nightly Update** workflow. The script:

1. Fetches **live odds** from the Odds API (spreads, totals, moneylines).
2. Resolves each game's teams to **KenPom** names and loads **KenPom ratings** (AdjO, AdjD, AdjT) from cache.
3. Optionally loads **today's KenPom FanMatch** from local parquet (scraped earlier in the pipeline).
4. For each game, computes **spread edge**, **moneyline edge**, and **over/under edge**.
5. Writes a single JSON object (to stdout and to `data/analysis/slate_today.json`) that is then pushed to Neon as `bracketiq_slate_today`.

The **consumer app** (BracketIQ API repo) only reads from Neon; it does not run the model.

---

## Inputs

| Input | Source | Used for |
|--------|--------|----------|
| **Live odds** | Odds API | Vegas spread (home POV), Vegas total, moneyline implied probs |
| **KenPom cache** | `app/data/cache/pomeroy_ratings_*.parquet` | AdjO, AdjD, AdjT per team; team list for name resolution |
| **Today's FanMatch** | `app/data/historical/fanmatch_2026.parquet` (today's rows) | KenPom predicted margin (home POV) and predicted total when game matches |
| **Historical FanMatch** | Same parquet (past dates) | Recency/schedule (optional, for recency adjustment) |

All spreads and margins are from the **home team's perspective**: negative spread = home favored, positive = away favored.

---

## Spread Edge

- **Vegas spread (home POV):** `vegas_spread`  
  - From Odds API: consensus spread for the **home** team only (negative = home favored).  
  - Example: Gonzaga -18.5 vs Oregon St → `vegas_spread = -18.5` when Gonzaga is home.

- **Model predicted margin (home POV):** `kenpom_predicted_margin_home_pov`  
  - **If the game is in today's FanMatch:** use KenPom's predicted margin from FanMatch, converted to home POV. For neutral-site games we try both (home, away) and (away, home) and negate when we match the reversed pair.  
  - **Else:** our formula:  
    - Efficiency margin per 100:  
      `(AdjO_h - AdjD_a) - (AdjO_a - AdjD_h) + HOME_COURT_ADVANTAGE`  
    - Point margin:  
      `our_margin = eff_margin_per_100 * (tempo / 100)`  
    - Then add **recency adjustment** (capped ±3 pts) and clamp to ±28.

- **Spread edge:**  
  `spread_edge = kenpom_predicted_margin_home_pov + vegas_spread`  
  - **Positive** → model has home winning by more than the spread → "Model likes HOME vs spread."  
  - **Negative** → model has home winning by less (or losing by more) → "Model likes AWAY vs spread."

- **Confidence:** `spread_edge_confidence` is derived from historical cover rates by |edge|:  
  NO_EDGE (<1), MILD (1–3), STRONG (3–5), HIGH_CONVICTION (≥5).

Games with missing or invalid **tempo** (AdjT) for either team are **skipped** (no fallback tempo).

---

## Moneyline Edge

- **Our model:**  
  - Same **our_margin** as above (efficiency × tempo/100 + recency, clamped).  
  - Win probability (home):  
    `model_win_prob_home = norm.cdf(our_margin / MARGIN_SIGMA)`  
  - `MARGIN_SIGMA = 11.5` (points).

- **Vegas:**  
  - `vegas_implied_prob_home` (and away) from consensus moneyline odds.

- **Moneyline edge:**  
  `moneyline_edge = model_win_prob_home - vegas_implied_prob_home`  
  - Positive → model likes HOME ML; negative → model likes AWAY ML.

---

## Over/Under Edge

- **Model total:**  
  - **If game is in FanMatch and we have a predicted total:** use KenPom's predicted total.  
  - **Else:**  
    `predicted_total = tempo * (AdjO_h + AdjO_a) / 100`

- **Over/under edge:**  
  `over_under_edge = predicted_total - vegas_total`  
  - Positive → model likes **OVER**; negative → model likes **UNDER**.

---

## Constants

| Constant | Value | Meaning |
|----------|--------|---------|
| `HOME_COURT_ADVANTAGE` | 5.0 | Points added to home in efficiency margin. |
| `MARGIN_SIGMA` | 11.5 | Scale for normal CDF converting margin → win prob. |
| `MAX_PREDICTED_MARGIN` | 28.0 | Clamp \|margin\| for edge/win prob. |
| `MAX_VEGAS_SPREAD_ABS` | 35.0 | Skip games with \|vegas_spread\| > this. |
| `RECENCY_WEIGHT` | 0.15 | Weight for recency vs expected margin. |
| `RECENCY_CAP_PTS` | 3.0 | Clamp recency adjustment to ± this. |

## Output JSON Structure

The model (and the Neon table) expose one object per game with fields like:

| Field | Type | Description |
|--------|------|-------------|
| `away_team`, `home_team` | string | Odds API names. |
| `away_team_kenpom`, `home_team_kenpom` | string | Resolved KenPom names. |
| `vegas_spread_home_pov` | number | Consensus spread, home POV (negative = home favored). |
| `vegas_total` | number | Consensus total (over/under). |
| `vegas_implied_prob_home`, `vegas_implied_prob_away` | number | From moneylines. |
| `spread_source` | string | `"kenpom_fanmatch"` or `"our_model"`. |
| `ou_source` | string | Same for total. |
| `kenpom_predicted_margin_home_pov` | number | Model margin (home POV) used for spread. |
| `kenpom_predicted_total` | number | Model total used for O/U. |
| `model_win_prob_home`, `kenpom_win_prob_home` | number | Same value (our model). |
| `spread_edge` | number | `margin + vegas_spread`. |
| `spread_edge_confidence` | string | NO_EDGE \| MILD \| STRONG \| HIGH_CONVICTION. |
| `moneyline_edge` | number | Model win prob (home) − Vegas implied (home). |
| `over_under_edge` | number | Model total − Vegas total. |
| `spread_edge_interpretation` | string | "Model likes HOME vs spread" or "Model likes AWAY vs spread". |
| `moneyline_edge_interpretation`, `ou_edge_interpretation` | string | Analogous. |
| `recency_adjustment_pts` | number | Points added to our_margin from recency. |
| `slow_underdog_flag`, `slow_underdog_note` | bool / string | Flag for slow-tempo underdogs. |

The full slate JSON also includes:

- **`all_games`:** list of all game objects (same as the rows in Neon).
- **`games_sorted_by_spread_edge`:** sorted by \|spread_edge\| then \|moneyline_edge\|.
- **`games_sorted_by_moneyline_edge`:** sorted by \|moneyline_edge\| then \|spread_edge\|.
- **`games_sorted_by_over_under_edge`:** sorted by \|over_under_edge\| then \|spread_edge\|.
- **`data_sources`:** short description of spread/O/U and moneyline sources.
- **`sanity_check`:** (when FanMatch exists) comparison of our margin/total vs KenPom for today.

---

## Getting a Copy of Today's JSON

- **From a live model run (this repo):**  
  From the **bracketiq** directory:  
  `python -m scripts.slate_today`  
  (Requires KenPom cache, ODDS_API_KEY, and optionally today's FanMatch.)  
  The script prints the full JSON to stdout and writes **`data/analysis/slate_today.json`**.

- **From Neon (consumer app):**  
  In the BracketIQ API repo, from the backend directory run **`py dump_slate_json.py`** with `NEON_DATABASE_URL` or `DB_URL` set. That writes **`slate_today_from_neon.json`** in the same shape.
